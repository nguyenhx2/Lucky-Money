## Admin Panelへのアクセス

**`Ctrl + Shift + G`**（Windows/Linux）または **`Cmd + Shift + G`**（macOS）で開きます。

**管理者パスワード**を入力してください（`.env.local`のSHA-256ハッシュ）。

---

## タブ: セットアップ

### 新しいセッションの作成

| フィールド | 説明 |
|----------|------|
| **予算** | 合計金額（VND）、千区切り入力対応 |
| **数量** | 封筒の数 |
| **開始時刻** | セッション開放時間（カウントダウン表示） |
| **最大選択数** | 1人あたりの最大抽選回数 |
| **リトライ%** | 追加ターンを自動付与するプレイヤーの割合 |
| **ボーナスラウンド** | シークレットボーナスラウンドの有効/無効 |
| **ボーナス%** | ボーナス封筒用に予約する予算の割合 |
| **音楽自動再生** | ON: ユーザー操作で再生。OFF: Playボタン押下が必要 |

### 再配布

未開封の封筒を再生成します。開封済みの封筒には影響しません。

---

## タブ: 履歴

- **カードビュー**: グリーティングカード、受取人名、金額
- **テーブルビュー**: 時系列リスト
- **統計**: 配布済み合計、残り封筒数、金額別内訳
- **メール再送**: 各参加者へ結果メールを手動送信

---

## タブ: リトライ（追加ターン）

特定のプレイヤーに追加抽選を付与：

1. 対象者のメールアドレスを入力
2. **「リトライ付与」**をクリック

> 重複付与はブロックされます（409エラー → フレンドリーなトーストメッセージ）。

**ボーナスラウンドの付与:**
1. ボーナス欄にメールアドレスを入力
2. **「ボーナス付与」**をクリック

> 1メールアドレスにつき1ボーナスグラント（DB unique制約）

---

## タブ: シミュレート

実際のアカウントを使わずにクレームフローをテスト：

1. テスト用メールアドレスを入力（DBに`[Test]`タグが付く）
2. リストから封筒を選択
3. **「シミュレート」**をクリック

---

## トースト通知

| 色 | 意味 |
|----|------|
| 🟢 緑 | 成功 |
| 🔴 赤 | エラー（具体的なメッセージ） |
| 🟡 黄 | 情報 |

HTTPステータス別メッセージ:
- **409** → 「すでに存在します」
- **404** → 「見つかりません」
- **403** → 「権限がありません」

---

## ベストプラクティス

1. **イベント前にセッションを作成** — まずシミュレートタブでテスト
2. **カウントダウンを設定**して参加者に開始時刻を明示
3. **メールホワイトリストを事前に一括追加**（CSV貼り付け可）
4. **ボーナスラウンド**: VIPまたはランダム抽選で事前に付与
5. **音楽自動再生OFF** → 静かなオフィス環境に適切
